Waar wordt de event listener gekoppeld aan de slider?

De event listener wordt ingesteld zodra de sliderwaarde wordt aangepast, opgevangen door het "input" en "change" event.

Waarom moeten we die op twee soorten events koppelen?

Om ervoor te zorgen dat de waarde van de slider correct wordt opgeslagen, moeten we luisteren naar zowel het slepen als het loslaten van de slider door de gebruiker.

In de CSS file wordt nergens een rode kleur opgegeven, waar wordt dan wel de rode kleur van het blokje ingesteld?

De rode kleur van het blokje wordt dynamisch aangepast via JavaScript door de inline stijl van de elementen met de klasse "colordemo" te wijzigen.

Waarom schrijven we telkens sliders[0] en colorDemos[0] en niet gewoon sliders of colorDemos?

We refereren telkens naar sliders[0] en colorDemos[0] omdat we werken met collecties van elementen. Door de index te gebruiken, krijgen we toegang tot het eerste element in deze collecties.

In de JavaScript-code passen we CSS-eigenschappen van een element aan. In de Chrome Developer Tools kun je de relevante CSS-regels voor een element bekijken onder het "Styles" tabblad.

Waar vind je de CSS-properties die programmatorisch werden ingesteld, zoals bijvoorbeeld de rode achtergrondkleur van het blokje?

Je kunt de programmatisch ingestelde CSS-properties vinden onder "body" > "styles" in de Chrome Developer Tools.
